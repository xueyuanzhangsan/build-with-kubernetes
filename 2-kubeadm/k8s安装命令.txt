证书续期（
kubeadm 默认证书为一年，一年过期后，会导致 api service 不可用，使用过程中会出
现：x509: certificate has expired or is not yet valid.
证书默认存放目录：/etc/kubernetes/pki
1.查询当前证书过期时间
kubeadm alpha certs check-expiration
也可以直接查看证书
cd /etc/kubernetes/pki
for i in `ll /etc/kubernetes/pki | grep crt | awk '{print $9}'`;do echo $i && openssl x509 -in $i -
noout -text |grep Not;done

2.备份
cp -R /etc/kubernetes/pki /etc/kubernetes/pki_bakup

3.续期证书
kubeadm alpha certs renew all --config=/root/cluster.yaml
也可以单个更新
更新下 kubeconfig 文件
Kubeadm config view >kubeadm.yaml
kubeadm init phase kubeconfig all --config kubeadm.yaml

.4.再次查看证书有效期
kubeadm alpha certs check-expiration

5.重新启动 kubelet

6.Node kubelet 过期
On master node
kubeadm init phase kubeconfig kubelet --node-name k18-5 --kubeconfig-dir /tmp/ --apiserver-advertise-address 192.168.56.5
scp /tmp/kubelet.conf k18-5:/etc/kubernetes/
systemctl restart kubelet
